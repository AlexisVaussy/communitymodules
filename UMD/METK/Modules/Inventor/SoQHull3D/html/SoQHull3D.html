SoQHull3D

Purpose

The SoQHull3D module calculates the convex hull of an 3d inventor scene using the "Quickhull Algorithm".

Usage

Connect an invetor scene to SoQHull3D and the module to a 3D viewer (e.g. SoExaminerViewer).
The left connector (out_qhull) gives you the convex hull, the right one (self) the input scene.

Details

None.

Inputs

An inventor scene.

Outputs

  • The convex hull as points (SoPointSet) and faces (SoIndexedFaceSet).
  • The input scene (SoQhull3D behaves like SoSeparator).

Parameters

Options

This panel contains drawing parameters and a state field.
  • Render Points / Point Color
    Select if points should be drawn.
    Select the point color.
    Default is enabled / blue.
  • Render Faces / Face Color
    Select if faces should be drawn.
    Select the face color.
    Default is enabled / yellow.
  • Use Colors
    Select if points and faces should be drawn in the defined colors.
    Deselect if points and faces should be drawn using inventor standard.
    Default is disabled.
  • State
    Gives some information about calculations.

Statistics

This panel contains statistical information about the convex hull.
  • # Input Values
    The number of vertex values that where collected from the inventor scene.
  • # Hull Vertices
    The number of vertices that build the convex hull.
  • # Hull Faces
    The number of faces that build the convex hull.
  • Surface
    The surface of the convex hull.
  • Volume
    The volume of the the convex hull.
The values for surface and volume are estimated by QHull functions.

"Hidden" options

These options are present in the automatic panel of SoQhull3D.
  • Show Normals
    Render the face or vertex normals of the convex hull.
    Default is disabled.
  • Use Face Normals
    Decide which kind of normals should be used for rendering: Face normals or vertex normals.
    Note: For some objects QHull does not generate vertex normals.
    Default is enabled.

Known Bugs

For some objects QHull does not generate vertex normals but the convex hull is computed. Use face normals instead.
