METK Messaging Modules

METKGlobalMessages / METKMsgSender / METKMsgReceiver

Alle Parameter von Objekten, Viewern, selektierte Objekte, Patientendaten etc. werden im ObjMgr gespeichert. Auf diesen kann jedes METK-Modul sowohl lesend als auch schreibend zugreifen. Durch das Event-System des ObjMgr ist jedes angeschlossene Modul in der Lage, über für es interessante Veränderungen von Parametern sofort informiert zu werden und darauf zu reagieren. Diese Art der Kommunikation erfolgt nicht synchronisiert, dass heißt Änderungen von Parametern werden gebroadcastet und es wird vom Sender keine Antwort erwartet. Es gibt jedoch einige spezielle Nachrichten, die alle Module betreffen, und Nachrichten, bei denen der Sender eine Bestätigung benötigt.

METKGlobalMessages

Beim Laden oder Schließen eines Falles werden Nachrichten versendet, auf die alle METK-Module hören. Mit Hilfe von METKGlobalMessages kann ein Makromodul auf Modulebene darüber informiert werden. In diesem Fall können die Module benötigte Daten erstellen bzw. löschen.

Details

Das Modul hört auf Änderungen des Eintrags [Communication/GlobalEvents/CaseLoaded]. Der Eintrag ist entweder "Loaded" oder "Cleanup". Wenn "Loaded" gesendet wurde, sind alle Einträge im ObjMgr bereits vorhanden, sodass auf den Daten gearbeitet werden kann.

Parameters

Loaded

Dieser Trigger wird benachrichtigt, wenn ein neuer Fall geladen wurde.

Cleanup

Dieser Trigger wird benachrichtigt, wenn ein Fall geschlossen werden soll.

METKMsgSender / METKMsgReceiver

Zur synchronisierten Kommunikation zwischen einzelnen Modulen wurden die Module METKMsgSender und METKMsgReceiver entwickelt. Diese beiden Module können über den ObjMgr sich gegenseitig Events zukommen lassen, deren Empfang bestätigt (processing) und deren Abarbeitung (done) bestätigt werden müssen. Es wird immer nur EIN Receiver benachrichtigt!!

Details

Die beiden Sender und Receiver können beliebige Nachrichten im String-Format miteinander austauschen. Über welchen "Kanal" die beiden Kommunizieren, wird über Message festgelegt. Sollte kein Receiver auf dem Kanal hören, wird die Übermittlung abgebrochen, d.h. Done ist "True" und Successful ist "False". Ist ein passender Receiver vorhanden, wird die Status auf "Processing" gesetzt. Der Inhalt der Nachricht wird über Data gesendet. Nach der gesendeten Bestätigung sind Done und Successful "True". Außer einer positiven Benachrichtigung können keine Nachrichten zurückgesendet werden.

Parameters

Message

Legt den Informationskanal fest.

Data

Enthält die zu sendende Nachricht.

Send / Done

Abschicken der Nachricht bzw. Bestätigen der Nachricht (nach dem Bearbeiten).

Status

Momentaner Status der Bearbeitung:

  • Done: keine Nachricht zu bearbeiten
  • Processing: Nachricht wird bearbeitet.
  • Nobody's listening: kein Receiver hört auf dem eingetragenen Kanal in Messages.

Done

Ist "False", wenn eine Nachricht gesendet wurde, aber noch keine Bestätigung zurückgekommen ist. Ist "True", wenn eine Bestätigung empfangen wurde oder kein passender Receiver vorhanden ist.

Successful

Ist nur dann "True", wenn die Nachricht erfolgreich bearbeitet wurde.
