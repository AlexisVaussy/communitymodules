AviWriter

Purpose

The AviWriter module can collect an arbitrary number of images via its image input and compress then into an AVI. The AVI is compressed by a user-selectable compression algorithm and is written onto disk.

Usage

Choose a filename, a suitable compression algorithm and provide the module's image input with an ML image. This image must have a z-extend of one, say it must be one slice. The datatype of the image must be MLuint8 or a compatible. The image may be a greyvalue image or a colored one with three or four color channels (RGB or RGBA). If the image is of the latter type (RGBA), the fourth alpha channel is simply ignored.
Also, the input image's width and height must be a multiple of four. This is due to the usage of the Windows vfw-API. Additionally, the image extends must not vary while recording or those particular frames are being ignored.

If the module's 'Enable Recording' is checked, any image input change (or notification) triggers the module to record a frame, which is immediatly inserted into the AVI. To finish a recording, press 'Finish Recording', in order to cancel a recording, press 'Reset Recording'.

On the 'Codec'-tab, there is an information field which displays the registered, available compression codecs on the current system. All codecs have unique identifiers consisting of four letters. If a codec is specified, only the first four letters are handled. Note that some of the displayed codecs are 'Read only' codecs, which means that one can play a movie that is compressed with a certain codec (in other words: the codec may be used for decompression), but this codec cannot be used for compressing images to a movie. In this case an error message is displayed in the status field and a default codec is chosen instead.

Details

The AviWriter makes use of the Windows vfw-API, where vfw stands for VideoForWindows. This brings forth that the module is available only under Windows.

Inputs

The input image must have a height and width which are multiples of four (4). The extend in z-dimension must be 1 (a sinlge slice). The datatype of the input image must be MLuint8 or compatible.
The color dimension can be 1 which results into a greyvalue movie. Or, the color dimension can have an extend of 3 or more, where the first three entries are interpreted as RGB values.

Outputs

The ouuput of the module is an AVi written onto disk.

Parameters

Main


Enable Recording
 If checked, all valid new images connected to the module's image input will be recorded. The frames are added on-the-fly into the current AVI movie. If unchecked, the module is set to passiv stance and no frames are recorded.
Output Filename
 The name of the output file. This must be specified before a recording is started.
Fps
 The number of frames per second of the target AVI movie.
Codec
 The currently used compression codec. This field is only an information field: the codec can be chosen on the 'Codec'-tab.
Num. Frames
 This information field displays the number of frames recorded so far into the current movie.
Reset Recording
 If clicked, the current recording of a movie is canceled and a frames are being discarded.
Finish AVI
 If clicked, the internal movie stream is closed. Beware: if the input images changes after a movie has been finished and 'Enable Recording' is still checked, a new movie is created under the old name and the old movie is lost.
Status
 Displays some status information and error messages.

Codec


Codec
 Here a codec code consisting of four letters can be entered. The codec is checked against the codecs installed on the current system and is accepted, if the codec is available. Addionally, the codec codes of all installed codecs are displayed in the field below.
Available Codecs
 Displays the codec codes of all installed codecs on the current system.

Events & Interaction

None.

Example

In the example network, adjust the default codec if necessary.
Otherwise simply increment the 'Start Slice' value on the SoView2D-panel. If satisfied, click 'Finish AVI' and have a look at the movie!
