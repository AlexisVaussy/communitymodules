AscendingAortaDetection

Purpose

The AscendingAorataDetection module allows to detect the ascending aorta within a computed tomography coronary angiography (CTCA) data set. With some parameter changes, it is also possible to detect the descending aorta. The module is based on the publication

Automatic Ascending Aorta Detection in CTA Datasets
Stefan C. Saur and Caroline Kühnel and Tobias Boskamp and Gábor Székely and Philippe Cattin
Bildverarbeitung für die Medizin 2008
Download paper in pdf format

Usage

Just connect the input image, adapt the parameters (per default, they are set to detect the ascending aorta) and press the Apply button to start the detection. As output, the world coordinates of a seed point within the ascending aorta is given. Further, all candidate points are given at the output outXMarkerList. The candidate points from largest cluster representing the ascending aorta are given at the output outXMarkerListLargestCluster whereas the candidates from the second largest cluster are given at outXMarkerListSecondLargestCluster.

Inputs

The module has one input image which will be internally casted to MLuint16type.

Outputs

The module has three XMarkerLists as output\
  • outXMarkerList: all candidate points as determined by the algorithm. Points from different clusters have a different XMarker type property.
  • outXMarkerListLargestCluster: all candidate points from the largest cluster.
  • outXMarkerListSecondLargestCluster: all candidate points from the second largest cluster.

Parameters

Main

  • Primary Threshold: primary threshold th_p in gray values.
  • Secondary Threshold: secondary threshold th_s in gray values.
  • Min Diameter: minimum diameter d_min in mm.
  • Max Diameter: maximum diameter d_max in mm.
  • Distance Cluster: distance limit for clustering d_clust.
  • Increment: increment of search ray delta_x, delta_y, delta_z.
  • Radio Limit: ratio a.
  • Use Ratio Limit: if checked, ratio criterion is applied in the detection process.
  • Start Search Area: start of search range VOI_x, VOI_y, VOI_z.
  • End Search Area: end of search range VOI_x, VOI_y, VOI_z.
  • Seed Point: seed point within the ascending aorta.
  • Run Time: run time of the algorithm in seconds.
  • Auto Apply: if checked, the algorithm is triggered upon connection of an input image.
  • Apply: manually run algorithm.
  • CommandList: List of fields that should be touched. One field per line in the form moduleName.fieldName.
  • Apply: Touch all fields one after the other as listed in CommandList.

LargestCluster

Statistical information about the detected largest cluster.
  • ClusterSize: size of cluster i.e. number of candidate points.
  • MeanIntensity: mean intensity within bounding box.
  • StdIntensity: std intensity within bounding box.
  • BaryCenter: barycenter of cluster.
  • LowerCenter: center point at lower extend (z smallest) of cluster.
  • UpperCenter: center point at upper extend (z largest) of cluster.
  • RadiusBaryCenter: radius at bary center of cluster.
  • BbLower: bounding box for cluster (see source code on how it is defined).
  • BbUpper: bounding box for cluster (see source code on how it is defined).
  • Extension: exentsion in z direction in mm.
  • MeanRadius: mean radius in mm of candidates points within cluster.
  • StdRadius: std of radius in mm of candidates points within cluster.

SecondLargestCluster

Statistical information about the detected second largest cluster (equivalent to the information for the largest cluster).
  • ClusterSize: size of cluster i.e. number of candidate points.
  • MeanIntensity: mean intensity within bounding box.
  • StdIntensity: std intensity within bounding box.
  • BaryCenter: barycenter of cluster.
  • LowerCenter: center point at lower extend (z smallest) of cluster.
  • UpperCenter: center point at upper extend (z largest) of cluster.
  • RadiusBaryCenter: radius at bary center of cluster.
  • BbLower: bounding box for cluster (see source code on how it is defined).
  • BbUpper: bounding box for cluster (see source code on how it is defined).
  • Extension: exentsion in z direction in mm.
  • MeanRadius: mean radius in mm of candidates points within cluster.
  • StdRadius: std of radius in mm of candidates points within cluster.

Info

  • Info: Textfield providing additional output information from the algorithm.
