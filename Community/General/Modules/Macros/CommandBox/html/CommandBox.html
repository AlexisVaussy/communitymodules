CommandBox

Purpose

The CommandBox macro allows to touch several buttons/fields one after the other. It is useful when several buttons within a network have to be touched each time to process it. To speed up, these buttons can be defined in a list and automatically be touched one after the other by just one mouse-click. Furthermore, there is an option to process all buttons multiple times in a loop - useful when running a network on a large number of data.

Usage

Write the fields to be touched into the CommandList. One field per line in the form moduleName.fieldName. For a comment (in a separate line), use the character #. Finally, press the Apply button to touch all the listed fields in your network.

Inputs

The module has no input.

Outputs

The module has no output.

Parameters

Main

  • CommandList: List of fields that should be touched. One field per line in the form moduleName.fieldName.
  • Apply: Touch all fields one after the other as listed in CommandList.

Parameters

  • Process events: If checked, the command MLAB.processEvents is executed after each field being touched.
  • Process inventor queue: If checked, the command MLAB.processInventorQueue is executed after each field being touched.
  • Log commands: if checked, the touched fields are logged into the console.
  • Sleep [ms]:: Idle time between touching two fields in milli seconds.

Loop

  • Do loop: If checked, the CommandList is processed multiple times, as defined by the loop parameters Start and Stop.
  • Start: Start value of the loop.
  • Stop: Stop value of the loop.
  • Value:: Current value of the loop (start Size <= value <= Stop).

Events & Interaction

None.

Example

The example network demonstrates the usage of the module by touching two different buttons of two Counter modules.
