PythonExec

Purpose

This macro enables you to run some python code from within the network. So you don't have to write a seperate script file. If you have more extensive code it's bad practice to use this module for that.

Usage

Simply write the python code you want to execute in the box on the Edit tab and press Run.

Details

On pressing Run this module wil call the Python exec() function with the specified command. It is possible to supply the command with several arguments. This is done by replacing the all occurences of $1$ in the given command by the string of the Arg1 field (and vice versa for $2$ and $3$), i.e. if Arg1 has the value 2.0 $1 will be replaced by '2.0' (including the quotes). So if you want the float value, you have to use float($1$).

Returning values is done in a similar maner: All occurences of %1% are returned to result1 and similar holds for %2%.

If you add nw = ctx.parent() to the command, you can use it just like the network attached python file except that you have to use nw. in stead of ctx. So the following command:

nw = ctx.parent()
print nw.field('PythonExec.command').value

will print the command itself when Run is pressed.
Since you have access to all network fields from within the command, you can also directly get arguments from the network or modify fields, e.g. :
nw = ctx.parent()
nw.field('PythonExec.result1').value = 'Directly modified by PythonExec'

To use a field from your network, you can simply drag and drop it on the command box. This will add a new line with the statement:

nw.field([fieldName]).value

Or if you drop a trigger field:

nw.field([fieldName]).touch()

If you drop the newValue field of a FieldIterator the following lines are added

for i in range( nw.field('[ModuleName].numValues').value ) :
nw.field('[ModuleName].curIndex').value = i
print nw.field('[ModuleName].curValue').stringValue()

This makes it easy to create a loop over all iterator values and perform some operations in between the iterations. This also holds for the FileIterator module.

Parameters

  • command The command(s) that should be executed
  • arg1 argument1
  • arg2 argument2
  • arg3 argument1
  • run Execute the command
  • onArgChange React to argument changes
  • result1 result1
  • result2 result2
  • comString Command as it is executes after replacing the argument strings
