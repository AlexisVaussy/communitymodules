BuildProjects

Purpose

Automatically (re)create project files and compile and link projects for a complete set of MeVisLab packages.
Warning: Only implemented for use on Windows with Visual Studio 2005. With some small changes, it can be used for other compilers in Windows as well. No support for Linux and Mac OS.

Usage

Select the root directory of the packages that you would like to build. Make sure that the compiler executable is set correctly and configure the way you would like to (re) build your projects. Warning: It does not check project dependencies.

Parameters

The builds can be configured globally, for all projects.

Directory
The root directory of the packages to build.
Default is D:/Documents/Mevislab/Packages/ISI.

Include Sub Directories
Toggle inclusion of subdirectories on/off.
Default is TRUE.

Recursion Depth
Set the depth of the subdirectory recursion, if applicable.
Default is -1.

Number Of Projects
Shows the total number of projects that was found under the root directory.
Default is 83.

Devenv Executable
The compiler executable. For VS2005 it is devenv.exe, for other compilers it might be a different name.
Default is C:/Program Files (x86)/Microsoft Visual Studio 8/Common7/IDE/devenv.exe.

Run
Start building the projects.

Project Files
Choose the mode for project file generation:
Skip: Do not create project files.
Create: Create the project file(s) only if they do not extist yet.
Recreate: Alsways (re)create the project file(s).
Default is Create.

Debug
Choose the mode for the debug version:
Skip: Do not build the debug vesrion.
Build: Build the debug version only if it is out-of-date or unexisting.
Rebuild: Alsways (re)build the debug version.
Default is Build.

Release
Choose the mode for the release version:
Skip: Do not build the release vesrion.
Build: Build the release version only if it is out-of-date or unexisting.
Rebuild: Alsways (re)build the release version.
Default is Build.

Progress
Shows the progress of the total build task.
